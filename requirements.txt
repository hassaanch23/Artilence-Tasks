# Version notes (a real compatibility story worth understanding):
#   * On x86_64 macOS, PyTorch's LAST published wheel is 2.2.2 -> we cap torch there.
#   * transformers 5.x requires torch >= 2.4, so we cap transformers < 5.
#   * torch 2.2.2 was built against NumPy 1.x, so we must keep numpy < 2.
torch>=2.2,<2.3         # backend that runs the model (x86_64 macOS caps at 2.2.2)
transformers>=4.44,<5   # model + pipeline API (5.x needs torch>=2.4)
numpy<2                 # torch 2.2.2 breaks on NumPy 2.x
sentencepiece>=0.1.99   # tokenizer used by T5 / flan-t5 models
